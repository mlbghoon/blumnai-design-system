# 타이포그래피 (Typography)

## 1. 텍스트 크기 규칙

**DS의 `size-*` 유틸리티 클래스를 사용한다.** Tailwind의 `text-xs`/`text-sm` 등은 사용하지 않는다.

| 용도 | DS 클래스 | 크기 | 비고 |
|------|-----------|------|------|
| 캡션, 뱃지, 메타데이터 | `size-xs` | 12px | 보조 정보 |
| 본문, 라벨, 목록 아이템 | `size-sm` | 14px | **기본값** (가장 많이 사용) |
| 섹션 제목, 강조 텍스트 | `size-md` | 16px | 소제목, 강조 |
| 페이지/패널 제목 | `size-lg` | 18px | 최상위 제목 |
| 큰 제목 (드물게) | `size-xl` | 20px | 특수 경우만 |

> 전체 DS 크기: `size-xs`(12) → `size-sm`(14) → `size-md`(16) → `size-lg`(18) → `size-xl`(20) → `size-2xl`(24) → `size-3xl`(30) ...

### 1-1. 선택 원칙 (권장이며 강제가 아님)

위 표는 **default 매핑**이다. 실제 선택은 픽셀이 아니라 **정보의 역할과 위계**로 판단한다.

#### 각 단계가 의미하는 역할

- `xs` — 본문에 종속된 보조/메타. 단독으로 읽지 않는 정보.
- `sm` — 사용자가 실제로 읽고 처리하는 기본 정보 단위.
- `md` — 영역/그룹의 입구. 스캔할 때 눈이 멈추는 지점.
- `lg` 이상 — 화면이나 뷰의 정체성을 알려주는 단일 앵커.

#### 판단 기준 (스스로에게 묻는다)

- 이 텍스트는 단독으로 의미가 성립하는가, 인접 텍스트의 부속인가? → 부속이면 한 단계 작게.
- 사용자가 정독하는가, 훑어보는가? → 훑는 정보면 한 단계 작게.
- 같은 영역 안에서 다른 텍스트와 위계 차이를 만들어야 하는가? → 한 단계 차이로 표현.
- 화면 전체에서 이 텍스트가 단 하나뿐인 앵커인가? → `lg` 이상 후보.

**위계는 절대 크기가 아니라 한 단계 차이로 만든다.** 같은 영역 안에서 인접 단계(sm↔md, md↔lg)로 대비를 주는 것이 우선이고, 두 단계 이상 점프는 '주인공'을 만들 때만 쓴다.

### 1-2. 권장에서 벗어나도 좋은 신호

다음 신호가 보이면 default를 벗어나는 것이 더 적절하다.

- **정보 밀도가 매우 높은 표/리스트** → 본문도 `xs`까지 내려갈 수 있다.
- **빈 상태·온보딩·결과 화면처럼 단일 메시지가 중심** → 본문이 `md`~`lg`로 올라갈 수 있다.
- **수치가 콘텐츠의 주인공인 카드(KPI, 가격, 카운터)** → 수치만 `2xl` 이상으로 점프하고 주변 라벨과 두 단계 이상 차이를 둔다.
- **권한·경고처럼 무게감이 필요한 라벨** → 같은 역할이어도 한 단계 위로 올린다.
- **밀도 높은 폼의 헬프 텍스트·검증 메시지** → `xs`로 내려 본문(`sm`)과 위계 분리.

## 2. 금지 패턴

```
❌ text-xs, text-sm, text-base, text-lg   → DS size-* 클래스 사용
❌ text-[10px], text-[11px], text-[13px]  → 임의 픽셀값 금지
❌ font-bold 남용                          → 아래 굵기 규칙 참조
```

## 3. 폰트 굵기 (font-weight)

3단계만 사용한다:

| 용도 | 클래스 | weight | 설명 |
|------|--------|--------|------|
| 일반 텍스트, 본문, 목록 | `font-normal` | **400** | 기본값 (생략 가능) |
| 버튼, Input 라벨, 폼 라벨 | `font-medium` | **500** | 버튼 텍스트 + 입력 필드 라벨 |
| 섹션 제목, 패널 제목, 강조 | `font-semibold` | **600** | 제목/헤딩 계열 |

```
❌ font-light (300)   → 사용 금지
❌ font-bold (700)    → 사용 금지 (font-semibold로 대체)
❌ font-extrabold     → 사용 금지
```

## 4. 텍스트 줄바꿈

| 속성 | Tailwind | 용도 | 제한 |
|------|----------|------|------|
| `text-wrap: balance` | `text-balance` | 제목, 짧은 텍스트 — 줄 간 길이 균등 분배 | Chromium 6줄, Firefox 10줄 이하 |
| `text-wrap: pretty` | `text-pretty` | 본문, 긴 텍스트 — 마지막 줄 orphan 방지 | 줄 수 제한 없음 |

## 5. 폰트 스무딩

> macOS에서 텍스트 렌더링을 선명하게 한다. **루트에 1회 적용** (개별 요소에 적용하지 않음).

```css
html {
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}
```

다른 플랫폼에서는 무시되므로 안전하게 전역 적용 가능.

## 6. Tabular Numbers

> 동적으로 변하는 숫자(카운터, 가격, 타이머)에는 `tabular-nums`를 적용하여 레이아웃 시프트를 방지한다.

```
✅ font-[tabular-nums]  → 카운터, 가격, 타이머 등 변동 숫자
❌ 정적 숫자에 tabular-nums → 불필요
```

## 7. 시맨틱 헤딩 (h1 단일성)

> **페이지당 `<h1>`은 단 1개만 사용한다.** 마케팅·분석 툴(GA, GTM, SEO 크롤러 등)이 페이지 식별과 구조 파싱에 `h1`을 기준점으로 사용하기 때문이다. 다중 `h1`은 추적·지표 집계를 깨뜨릴 수 있다.

### 7-1. 원칙

- 한 페이지의 `<h1>`은 그 페이지가 무엇인지 알려주는 **하나의 정체성**이다.
- 시각적으로 큰 제목이 여러 개 필요해도 마크업 위계는 `<h2>` 이하로 내려 쓴다.
- **시각적 강조와 시맨틱 위계는 분리한다.** '크게 보이는 것'과 '문서 구조의 1순위'는 다르다 — 크기는 DS `size-*`로 자유롭게 맞추되, 태그는 문서 구조를 따른다.
- 모달·드로어·임베드된 위젯 등 페이지 안의 보조 영역은 절대 `<h1>`을 쓰지 않는다.

```
❌ <h1>대시보드</h1> ... <h1>최근 활동</h1>           → 다중 h1 금지
❌ <h1 class="size-md">강조 텍스트</h1>               → 시각 강조 목적의 h1 금지
✅ <h1>대시보드</h1> ... <h2>최근 활동</h2>           → 페이지당 h1 1개
✅ <h2 class="size-xl font-semibold">큰 섹션</h2>     → 크게 보여야 하면 크기로, 태그는 h2 이하
```

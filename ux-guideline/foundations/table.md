# 데이터 테이블 (Data Table)

## 1. 기본 원칙

- 정형화된 열(column) 기반 데이터는 **DS DataGrid 컴포넌트**를 사용한다.
- 행 높이는 **고정이 아닌 가변**이다. 최소 높이를 유지하되, 내용이 많으면 늘어난다.
- 셀 텍스트는 기본적으로 **줄바꿈 허용**. 데이터 가독성이 말줄임보다 우선한다.

## 2. 행 높이

### 최소 높이 + 가변

```
min-height: 32px     (고정 아님 — 최소 높이)
padding-y: 6px       (상하 여백 유지)
height: auto         (내용에 따라 늘어남)
```

내용이 1줄이면 32px, 여러 줄이면 **상하 패딩을 유지한 채** 높이가 자연스럽게 증가한다.

```
❌ height: 32px (고정)        → 긴 텍스트가 잘리거나 넘침
✅ min-height: 32px (가변)    → 최소 높이 보장 + 내용 확장
```

### 현재 DS 상태 (엔지니어 수정 필요)

> 현재 DataGrid는 `height: 32px` 고정 + `truncate`(말줄임)로 구현되어 있다.
> 아래 변경이 필요하다:
>
> - DataGridCell: `height` → `min-height` + `padding-y`
> - CellText: `truncate` → 기본 줄바꿈 허용 (말줄임은 옵션)
> - 가상화(virtualization) 사용 시: `getRowHeight`로 동적 높이 계산 필요

## 3. 셀 텍스트 처리

| 모드 | 동작 | 용도 |
|------|------|------|
| **wrap** (기본) | 줄바꿈 허용, 행 높이 가변 | 대부분의 텍스트 셀 — 내용 전체 표시 |
| **truncate** | 말줄임(...), 행 높이 고정 | ID, 날짜, 상태값 등 짧은 고정 데이터 |

- 기본은 **wrap**. 데이터를 잘라서 보여주면 사용자가 내용을 파악할 수 없다.
- **truncate**는 내용이 짧고 예측 가능한 열에만 사용한다.
- truncate 셀에는 **TooltipTrigger**로 전체 텍스트 확인 수단을 제공한다.

## 4. 열 너비

| 전략 | 설명 | 용도 |
|------|------|------|
| **고정 너비** | `width: Npx` | ID, 체크박스, 상태, 날짜 등 길이가 예측 가능한 열 |
| **비율 너비** | `flex: N` 또는 `minWidth + flex` | 이름, 설명, 메모 등 가변 텍스트 열 |

- 가변 텍스트 열은 **최소 너비(`minWidth`) + flex**로 설정하여 내용이 너무 좁아지지 않도록 한다.

## 5. 셀 구분선

| 구분선 | 기본 | 용도 |
|--------|------|------|
| **가로 (border-bottom)** | 있음 | 행 간 구분 — 항상 유지 |
| **세로 (border-right)** | **없음** | 열 간 구분 — 기본적으로 표시하지 않음 |

- 세로 구분선은 **기본 OFF**. 열 간 간격과 정렬만으로 충분히 구분된다.
- 데이터 밀도가 높거나 열 간 경계가 모호한 경우에만 **옵션으로 ON**.

```
✅ 가로선만 (기본)          → 깔끔하고 현대적인 테이블
✅ 가로 + 세로 (옵션)       → 밀도 높은 스프레드시트형
❌ 세로선 항상 표시          → 시각적 잡음, 기본값으로 부적절
```

### 현재 DS 상태 (엔지니어 수정 필요)

> 현재 DataGrid는 `border-r-default`가 셀/헤더에 하드코딩되어 세로 구분선이 항상 표시된다.
> 세로 구분선을 제어하는 prop(예: `columnBorder?: boolean`)이 필요하며, 기본값은 `false`여야 한다.
>
> - DataGridCell: `border-r-default` → prop 기반 조건부 적용
> - DataGridHeader: 동일
> - DataGridLoading: 동일

## 6. 밀도 (Density)

| 밀도 | 최소 행 높이 | 상하 패딩 | 용도 |
|------|-------------|----------|------|
| **기본** | 32px | 6px | 대부분의 테이블 |
| **compact** | 28px | 4px | 대량 데이터, 대시보드 |
| **comfortable** | 40px | 10px | 적은 행, 강조 필요 |

> 기본 밀도를 우선 사용하고, 데이터 양과 화면 맥락에 따라 조정한다.

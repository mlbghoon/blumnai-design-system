# 추후 개선 목록

UI Sample.tsx 작업 중 발견한, 디자인 시스템·가이드라인에 추후 반영하면 좋을 항목.

---

## 1. 시맨틱 soft 배경 토큰 비대칭

**현황**
- `bg-state-destructive-soft`만 존재
- `bg-state-informative-soft`, `bg-state-warning-soft`, `bg-state-success-soft`는 정의되어 있지 않음
- `border-{informative|warning|success|destructive}` 와 `text-{informative|warning|success|destructive}` 는 4종 모두 존재 → border/text는 대칭, 배경만 비대칭

**문제**
- 정보·경고·성공 톤의 콜아웃을 만들 때 시맨틱 soft 배경 패턴을 쓸 수 없다
- DS `InfoBox` 컴포넌트가 있어 우회는 가능하지만, 인라인 배지/태그/하이라이트 등 InfoBox로 감싸기 어색한 케이스에서는 대안이 없다

**참고 위치**
- 토큰 정의: [src/styles/tokens/colors.css](../src/styles/tokens/colors.css)
- 유틸리티: [src/styles/utilities.css](../src/styles/utilities.css)

**제안**
- `--bg-state-informative-soft`, `--bg-state-warning-soft`, `--bg-state-success-soft` 토큰 추가
- 4개 테마(Light-A, Dark, Theme-B Light, Theme-B Dark) 모두에 대응

---

## 2. Codex 샘플의 무효 클래스

**현황**
- [UI Sample.tsx](UI%20Sample.tsx) Codex Freeform 섹션에서 `bg-state-informative-soft` 클래스를 사용 중
- 해당 토큰/유틸리티가 DS에 정의되어 있지 않아 실제 렌더 시 배경이 적용되지 않음 (transparent fallback)

**참고**
- 아이콘 박스 두 곳에서 사용 (Microsoft OneDrive 카드의 ☁ 박스)

**제안**
- 1번 항목과 함께 처리: 토큰이 추가되면 자동 해결
- 그 전까지 단기 수정이 필요하면 `bg-subtle` 또는 `bg-default` + `border-default`로 임시 교체

---

## 3. KPI 증감 시그널 아이콘

**현황**
- Claude Freeform 섹션의 KPI 카드 증감 표시는 유니코드 `▲▼` + `text-success`/`text-destructive`로 처리
- DS `Icon` 컴포넌트의 trend up/down 아이콘이 있는지 확인 후 일관화 가능

**제안**
- DS `Icon` 카탈로그에서 trend/arrow up·down 아이콘이 있다면 KPI 증감 시그널 표준 패턴으로 가이드라인에 추가
- 없다면 시스템·화살표 아이콘 셋 점검 후 보완

---

## 4. (선택) UI Sample.tsx 패턴 카탈로그 확장 후보

UI Sample.tsx에 아직 없는 complex UI 패턴 중, AI가 자주 만들게 될 화면 유형:

- KPI 카드 + 증감 시그널 (이번에 추가됨)
- 시계열 + 분포 결합 분석 카드 (이번에 추가됨)
- DS InfoBox 변형(info/warning/success) 카드 안 배치 (이번에 추가됨)
- Empty state 패턴 (DS empty-state 컴포넌트 활용)
- Wizard step header / 단계 진행 표시
- Notification stream 리스트
- AI 채팅 응답 카드 (요약 + 액션)

이번 작업에서 세 가지는 채워졌으므로, 나머지는 후속 PR로 다룰 수 있는 후보.

# 케이스 02: 설정/폼 화면

## 화면 구조

```
┌─────────────────────────────────────────┐
│ 설정                                    │
├─────────────────────────────────────────┤
│                                         │
│ 프로필 정보                              │
│ ┌─────────────────────────────────────┐ │
│ │ 이름     [___________________]      │ │
│ │ 이메일   [___________________]      │ │
│ │ 전화번호 [___________________]      │ │
│ └─────────────────────────────────────┘ │
│                                         │
│ 알림 설정                                │
│ ┌─────────────────────────────────────┐ │
│ │ 이메일 알림          [ON/OFF]       │ │
│ │ 푸시 알림            [ON/OFF]       │ │
│ │ 마케팅 수신          [ON/OFF]       │ │
│ └─────────────────────────────────────┘ │
│                                         │
│                    [취소]  [저장]        │  ← secondary, primary
└─────────────────────────────────────────┘
```

## Primary 배치 판단

| 위치 | 역할 | 스타일 | 이유 |
|------|------|--------|------|
| [저장] | 변경사항 최종 저장 | `primary` | 이 화면의 유일한 핵심 CTA |
| [취소] | 변경 취소 | `secondary` | 보조 액션 |
| [ON/OFF] | 토글 스위치 | DS Switch 컴포넌트 | 인라인 설정, 버튼 아님 |

- 전체 화면 primary 합계: **1개** (저장 버튼만)

## 폼 레이아웃 규칙

- 섹션 제목: `size-md` + `font-semibold`
- 라벨: `size-sm` + `font-medium`
- 입력 필드: DS Input 컴포넌트 사용
- 섹션 간 구분: 간격으로 구분 (border 불필요)
- 액션 버튼: 폼 하단 우측 정렬

## 흔한 실수

- 각 섹션마다 개별 [저장] primary 버튼을 넣는 것 → 최하단에 1개만
- [초기화] 버튼을 primary로 만드는 것 → `ghost` 또는 `secondary`
- 토글/체크박스 옆에 primary 색상 라벨을 넣는 것 → 라벨은 `text-default`
- 필수 항목 표시에 primary 색상을 쓰는 것 → 별표(`*`)는 destructive 색상

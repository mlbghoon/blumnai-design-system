## Primary 배치 판단

| 위치 | 역할 | 스타일 | 이유 |
|------|------|--------|------|
| [새 프로젝트] | 핵심 생성 액션 | `primary` | 대시보드의 유일한 핵심 CTA |
| 통계 카드 숫자 | 데이터 표시 | `text-default` | 숫자 자체가 정보, 강조 불필요 |
| [전체 보기] | 네비게이션 | `ghost` 또는 링크 | 보조 탐색 |
| [리포트 보기] | 네비게이션 | `ghost` 또는 링크 | 보조 탐색 |

- 헤더(컨테이너 ①)에 primary 1개: [새 프로젝트]
- 하위 카드들에는 primary 없음
- 전체 화면 primary 합계: **1개**

## 카드 구성 규칙

### 통계 카드 (상단)
- 플랫 카드: `bg-default` 또는 `bg-subtle`, shadow 없음
- 숫자 강조: `size-2xl` + `font-semibold` + `text-default`
- 라벨: `size-sm` + `text-subtle`
- 숫자에 primary 색상을 쓰지 않는다

### 콘텐츠 카드 (하단)
- 강조 카드: `rounded-lg` + `border-default` + `shadow-component-default`
- 카드 내부 액션: `ghost` 또는 텍스트 링크
- 카드 제목: `size-md` + `font-semibold`

## 흔한 실수

- 통계 카드의 숫자를 primary 색상으로 강조하는 것 → `text-default`로 충분
- 각 카드마다 [자세히 보기] primary 버튼을 넣는 것 → 카드 4개면 primary 4개, 규칙 위반
- "진행 중" 숫자만 primary로 강조하는 것 → 상태 구분은 아이콘이나 뱃지 색상으로
- 대시보드에 primary 링크를 여러 개 사용하는 것 → 링크는 `text-subtle` + 밑줄 또는 `ghost` 버튼

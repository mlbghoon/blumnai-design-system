# 케이스 04: 빈 상태 (Empty State)

## 화면 구조

### 유형 A: 단일 CTA

```
┌─────────────────────────────────────┐
│                                     │
│          (일러스트/아이콘)            │
│                                     │
│                                     │
│       아직 등록된 항목이 없습니다       │
│    항목을 추가하여 시작해 보세요       │
│                                     │
│           [항목 추가하기]             │  ← primary
│                                     │
└─────────────────────────────────────┘
```

### 유형 B: 주요 + 보조 CTA

```
┌─────────────────────────────────────┐
│                                     │
│          (일러스트/아이콘)            │
│                                     │
│      검색 결과가 없습니다             │
│   다른 키워드로 검색하거나             │
│   새 항목을 등록해 보세요             │
│                                     │
│     [검색 초기화]  [새 항목 등록]     │  ← secondary, primary
│                                     │
└─────────────────────────────────────┘
```

## Primary 배치 판단

| 위치 | 역할 | 스타일 | 이유 |
|------|------|--------|------|
| [항목 추가하기] | 핵심 전환 액션 | `primary` | 빈 상태를 벗어나는 유일한 CTA |
| [새 항목 등록] | 핵심 전환 액션 | `primary` | 빈 상태를 벗어나는 주요 CTA |
| [검색 초기화] | 보조 액션 | `secondary` | 대안 경로 |

- 빈 상태 전체가 1개 컨테이너 → primary 최대 1개
- 전체 화면 primary 합계: **1개**

## 일러스트/아이콘 규칙

빈 화면에서는 장식요소는 isometric icon 사용을 우선으로 한다.

| 속성 | 값 | 비고 |
|------|-----|------|
| View | top (위에서 내려다본 시점) | 입체감 없이 평면적 |
| Size | `48px` (기본) / `64px` (강조) | 과도한 크기 지양 |
| Stroke color | `isometric icon--stroke-color` | primary 색상 사용 금지 |
| Position | 수평·수직 중앙 (middle) | 제목 바로 위 배치 |

- 일러스트/아이콘은 항상 **stroke 기반**(line icon)으로 사용
- 배경 fill 없이 stroke만으로 표현 → 빈 상태의 가벼운 톤 유지
- 48px: 일반 빈 상태, 64px: 전체 페이지 빈 상태 등 강조가 필요한 경우

## 구성 규칙

- 제목: `size-md` + `font-semibold` + `text-default`
- 설명: `size-sm` + `text-subtle`
- CTA: 설명 아래 중앙 정렬
- 아이콘/일러스트에 primary 색상 사용 금지 → `isometric icon--stroke-color`

## 흔한 실수

- 일러스트를 primary 색상으로 칠하는 것 → 일러스트는 `text-muted` 계열 중립색
- [항목 추가] + [가이드 보기] 두 개를 모두 primary로 만드는 것 → 보조는 `secondary`
- 빈 상태인데 상단 헤더에도 [+ 추가] primary가 있는 것 → 빈 상태의 CTA와 중복, 하나만 노출하거나 헤더 쪽을 `secondary`로 낮춘다

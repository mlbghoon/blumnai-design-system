# UI 케이스 레퍼런스

AI가 화면을 생성할 때 참고하는 구체적 UI 구성 예시 모음.
`일관성-가이드라인.md`의 규칙을 실제 화면 단위로 풀어낸 것이다.

## 케이스 목록

| 파일 | 화면 유형 | 핵심 포인트 |
|------|-----------|-------------|
| [01-리스트-상세.md](01-리스트-상세.md) | 리스트 + 상세 화면 | primary CTA 배치, 리스트 구분 방식 |
| [02-설정-폼.md](02-설정-폼.md) | 설정/폼 화면 | primary는 최종 저장에만, 폼 레이아웃 |
| [03-대시보드.md](03-대시보드.md) | 대시보드 | 카드 간 primary 위계 분배 |
| [04-빈-상태.md](04-빈-상태.md) | Empty State | CTA 위계, 일러스트 배치 |
| [05-모달-다이얼로그.md](05-모달-다이얼로그.md) | 모달/다이얼로그 | 확인/취소 버튼 위계 |
| [06-타이틀.md](06-타이틀.md) | 타이틀 (참고용) | PageTitle vs CardTitle 구분 |

## 사용 방법

- 새 화면을 만들 때 가장 유사한 케이스를 먼저 참고한다
- 케이스는 "이렇게 해라"가 아니라 "이런 위계로 판단해라"를 보여주는 것이다
- 케이스에 없는 화면 유형은 가장 가까운 케이스를 기준으로 응용한다
